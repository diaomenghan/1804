案例1:为虚拟机 server 配置以下静态地址参数
C 主机名:server0.example.com
C IP地址:172.25.0.11
C 子网掩码:255.255.255.0
C 默认网关:172.25.0.254
C DNS服务器:172.25.254.254
[root@server0 ~]# echo server0.example.com > /etc/hostname
[root@server0 ~]# nmcli connection show  
[root@server0 ~]# nmcli connection modify 'System eth0' ipv4.method manual ipv4.addresses '172.25.0.11/24 172.25.0.254' connection.autoconnect yes
[root@server0 ~]# nmcli connection up 'System eth0' 
[root@server0 ~]# echo nameserver 172.25.254.254 > /etc/resolv.conf 


案例2:虚拟机 server0上操作
        新建用户 alex，其用户ID为3456，密码是abcd
	创建下列用户、组以及组的成员关系： 
	 C 一个名为 adminuser 的组 
	 C 一个名为 natasha 的用户，其属于 adminuser 组， 这个组是该用户的从属组 
	 C 一个名为 harry 的用户，其属于 adminuser 组，这个 组是该用户的从属组 
	 C 一个名为 sarah 的用户，其在系统中没有可交互的 Shell（/sbin/nologin），并且不是 adminuser 组的成员 
	 C natasha 、harry、sarah 的密码都要设置为 redhat
[root@server0 ~]# useradd -u 3456 alex
[root@server0 ~]# echo abcd | passwd --stdin alex
[root@server0 ~]# groupadd adminuser
[root@server0 ~]# useradd -G adminuser natasha
[root@server0 ~]# useradd -G adminuser harry
[root@server0 ~]# useradd -s /sbin/nologin sarah
[root@server0 ~]# echo redhat | passwd --stdin natasha
[root@server0 ~]# echo redhat | passwd --stdin harry
[root@server0 ~]# echo redhat | passwd --stdin sarah


案例3:虚拟机 server0上操作,tar制作/释放归档压缩包（zcf、ztf、zxf、jcf、jtf、jxf、cf、tf）
    1）备份/boot、/home这两个文件夹，保存为boothome.tar.gz文件
    2）查看boothome.tar.gz文件内包含哪些内容 
    3）将boothome.tar.gz释放到文件夹/root/boothome/下
    4）创建一个名为 /root/backup.tar.bz2 的归档文件，其中包含 /usr/local 目录中的内容
[root@server0 ~]# tar -zcPf boothome.tar.gz /boot/ /home/
[root@server0 ~]# tar -tf boothome.tar.gz 
[root@server0 ~]# mkdir /root/boothome
[root@server0 ~]# tar -xf boothome.tar.gz -C  /root/boothome
[root@server0 ~]# tar -jcPf /root/backup.tar.bz2 /usr/local/

案例5:配置Samba文件夹共享，防火墙默认区域设置为trusted，修改SElinux状态为Enforcing
在 server0 上发布共享文件夹
 1）此服务器必须是 STAFF 工作组的一个成员 
 2）发布目录 /devops，共享名为 devops 
 5）用户 harry 对共享 devops可读可写，密码是 123
[root@server0 ~]# vim /etc/selinux/config 
[root@server0 ~]# yum -y install samba
[root@server0 ~]# vim /etc/samba/smb.conf 
 workgroup = STAFF
[devops]
path = /devops
write list = harry
[root@server0 ~]# pdbedit -a harry
[root@server0 ~]# mkdir /devops
[root@server0 ~]# setfacl -m u:harry:rwx /devops
[root@server0 ~]# getsebool -a | grep samba
[root@server0 ~]# setsebool -P samba_export_all_rw on
[root@server0 ~]# systemctl restart smb
[root@server0 ~]# systemctl enable smb

在 desktop0 上访问共享文件夹

 1）访问 server0 上发布共享文件夹,实现开机自动挂载到/mnt/smb
 2）实现对/mnt/smb目录可以读和写
[root@desktop0 ~]# yum -y install cifs-utils
[root@desktop0 ~]# mkdir /mnt/smb
[root@desktop0 ~]# vim /etc/fstab 
[root@desktop0 smb]# tail -1 /etc/fstab 
//172.25.0.11/devops /mnt/smb cifs defaults,user=harry,pass=123,_netdev 0 0
[root@desktop0 ~]# mount -a
[root@desktop0 ~]# df -h
[root@desktop0 ~]# cd /mnt/smb/
[root@desktop0 smb]# ls
[root@desktop0 smb]# touch 123123
[root@desktop0 smb]# ls



案例6：普通NFS共享的实现
  1. 在 server0 上配置NFS服务 
   1）只读的方式共享目录 /public，只能被172.25.0.0/24网段中的客户机系统访问 
[root@server0 devops]# yum -y install nfs-utils
[root@server0 devops]# vim /etc/exports
[root@server0 devops]# cat /etc/exports
/public 172.25.0.0/24(ro)
[root@server0 devops]# mkdir /public
[root@server0 devops]# touch /public/qeadad
[root@server0 devops]# systemctl restart nfs-server
[root@server0 devops]# systemctl enable nfs-server

  2. 在 desktop0 上访问NFS共享目录 
   1）将 server0 的 /public 挂到本地 /mnt/nfsmount 
   2）这些文件系统在系统启动时自动挂
[root@desktop0 ~]# showmount -e 172.25.0.11
/public 172.25.0.0/24
[root@desktop0 ~]# mkdir /mnt/nfsmount
[root@desktop0 ~]# vim /etc/fstab 
[root@desktop0 ~]# tail -1 /etc/fstab
172.25.0.11:/public /mnt/nfsmount nfs defaults,_netdev 0 0
[root@desktop0 ~]# mount -a
[root@desktop0 ~]# df -h
[root@desktop0 ~]# cd /mnt/nfsmount/
[root@desktop0 nfsmount]# ls
qeadad
[root@desktop0 nfsmount]# touch 123
touch: 无法创建"123": 只读文件系统
